# Руководство по решению проблем (Актуальные исправления)

## Критические проблемы и их решения

### 1. "BleakOutOfConnectionSlotsError" - Ошибка Bluetooth соединений

**Проблема:** `Failed to connect after 9 attempt(s): No backend with an available connection slot`

**Причина:** Bluetooth адаптер исчерпал лимит одновременных подключений.

**Решение:**
- ✅ **Используйте агрессивное отключение соединений** (включено по умолчанию в новой версии)
- ✅ **Перезагрузите Bluetooth адаптер:** `sudo systemctl restart bluetooth`
- ✅ **Увеличьте интервал опроса** в настройках интеграции (рекомендуется 30-60 секунд)
- ✅ **Проверьте, что нет других Bluetooth устройств**, использующих много соединений
- ✅ **Перезагрузите HomeAssistant** после установки интеграции

### 2. "Auth failed" - Ошибка аутентификации

**Проблема:** Ошибка аутентификации при подключении

**Решение:**
- ✅ **Убедитесь, что мультиварка включена и в режиме пары** (индикатор мигает)
- ✅ **Попробуйте стандартный ключ:** `[0x00, 0x01, 0x02, 0x03, 0x04, 0x05, 0x06, 0x07]`
- ✅ **Перезагрузите мультиварку** и попробуйте снова
- ✅ **Проверьте расстояние до устройства** (не более 10 метров)
- ✅ **Удерживайте кнопку питания 3-5 секунд** для включения режима пары

### 3. "Device not found" / "NoneType object has no attribute 'name'"

**Проблема:** Мультиварка не обнаруживается при сканировании

**Решение:**
- ✅ **Убедитесь, что Bluetooth включен** на сервере HomeAssistant
- ✅ **Проверьте, что мультиварка включена и в режиме пары**
- ✅ **Попробуйте перезагрузить Bluetooth адаптер**
- ✅ **Проверьте расстояние до устройства**
- ✅ **Убедитесь, что мультиварка поддерживается** (RMC-M40S и другие модели серии RMC)
- ✅ **Проверьте MAC-адрес** - он должен быть в формате AA:BB:CC:DD:EE:FF

### 4. "Connection lost" - Потеря соединения

**Проблема:** Соединение с мультиваркой теряется

**Решение:**
- ✅ **Проверьте расстояние до устройства**
- ✅ **Убедитесь, что нет помех** от других Bluetooth устройств
- ✅ **Попробуйте уменьшить интервал опроса**
- ✅ **Перезагрузите мультиварку**
- ✅ **Используйте persistent=False** в настройках для агрессивного отключения

### 5. "Command timeout" - Таймаут команд

**Проблема:** Таймаут при выполнении команд

**Решение:**
- ✅ **Проверьте расстояние до устройства**
- ✅ **Убедитесь, что мультиварка включена**
- ✅ **Попробуйте перезагрузить мультиварку**
- ✅ **Проверьте, что мультиварка не занята** другой операцией
- ✅ **Увеличьте BLE_RECV_TIMEOUT** в коде (если необходимо)

### 6. "RuntimeWarning: coroutine 'CookerConnection._disconnect' was never awaited"

**Проблема:** Предупреждение о неожиданной корутине

**Решение:**
- ✅ **Обновите интеграцию** - эта проблема исправлена в последней версии
- ✅ **Перезагрузите HomeAssistant** после обновления
- ✅ **Проверьте, что используется async/await** для всех корутин

## Диагностика

```bash

sudo systemctl status bluetooth

# Перезагрузка Bluetooth

# Сканирование устройств
```

## Настройка Bluetooth

### Для Raspberry Pi

```bash
# Увеличение лимита соединений
sudo nano /etc/bluetooth/main.conf

# Добавьте в секцию [Policy]
AutoEnable=true

# Перезапустите Bluetooth
sudo systemctl restart bluetooth
```

### Для других систем

```bash
# Проверка версии BlueZ
bluetoothctl --version

# Обновление BlueZ при необходимости
sudo apt update && sudo apt upgrade bluez

# Проверка поддержки Bluetooth
hciconfig
```

### Для систем с ограниченными Bluetooth ресурсами

```bash
# Установка параметров для ограниченных систем
sudo nano /etc/systemd/system/bluetooth.service.d/override.conf

# Добавьте:
[Service]
ExecStart=
ExecStart=/usr/lib/bluetooth/bluetoothd --compat

# Перезагрузите систему
sudo systemctl daemon-reload
sudo systemctl restart bluetooth
```

## Технические детали исправлений

### Исправленные проблемы в последней версии:

1. **RuntimeWarning: coroutine 'CookerConnection._disconnect' was never awaited**
   - **Причина:** Метод `stop()` не был асинхронным
   - **Исправление:** Сделан метод `async def stop()` и добавлен `await self._disconnect()`

2. **BleakOutOfConnectionSlotsError**
   - **Причина:** Слишком много одновременных Bluetooth соединений
   - **Исправление:** Включено агрессивное отключение соединений, улучшена обработка ошибок

3. **NoneType object has no attribute 'name'**
   - **Причина:** Попытка доступа к несуществующему устройству
   - **Исправление:** Добавлена проверка `if self._device is None` и улучшенная обработка ошибок

4. **Проблемы с аутентификацией**
   - **Причина:** Нестабильное соединение во время аутентификации
   - **Исправление:** Улучшена обработка ошибок аутентификации и автоматическое переподключение

### Рекомендуемые настройки для стабильной работы:

```yaml
# Оптимальные настройки для большинства систем
skycooker:
  scan_interval: 30-60  # Интервал опроса
  persistent: false    # Агрессивное отключение для экономии ресурсов
  device: auto         # Автоматический выбор адаптера
```

Эти исправления должны решить основные проблемы с подключением и стабильностью работы интеграции.
