# SkyCooker Integration for Home Assistant

Интеграция для управления мультиварками Redmond серии RMC через Bluetooth.

## Поддерживаемые модели

- **RMC-M40S** (основная поддержка)
- **RMC-M42S** (экспериментальная поддержка)

## Требования

- Home Assistant 2025.12.5+
- Bluetooth адаптер
- Мультиварка Redmond RMC-M40S в режиме Bluetooth-паринга

## Установка

1. Скачайте интеграцию в папку `custom_components/skycooker`
2. Перезагрузите Home Assistant
3. Перейдите в Настройки → Интеграции → Добавить интеграцию
4. Выберите "SkyCooker"
5. Введите MAC-адрес и пароль устройства

## Настройка

### MAC-адрес устройства
Найдите MAC-адрес вашей мультиварки в настройках Bluetooth вашего устройства.

### Пароль
Пароль должен быть в HEX формате, длиной 8 байт. Вы можете:
- Использовать генератор: https://crypt-online.ru/crypts/text2hex/
- Или попробовать стандартные пароли: `0000000000000000`, `1111111111111111`

### Режим сопряжения
1. Включите мультиварку
2. Удерживайте кнопку сопряжения до появления мигающего индикатора
3. Начните процесс добавления интеграции в Home Assistant

## Возможности

- Включение/выключение мультиварки
- Управление режимами приготовления
- Установка температуры
- Контроль времени приготовления
- Мониторинг состояния

- Stewing (Тушение)

- Steaming (На пару)

## Диагностика проблем

### Проблема: "Basic connection test failed"
**Решение:**
1. Убедитесь, что мультиварка в режиме Bluetooth-паринга
2. Проверьте расстояние (не более 3-5 метров)
3. Перезагрузите мультиварку и Home Assistant
4. Попробуйте использовать ESPHome Bluetooth proxy для лучшей стабильности

### Проблема: "Device not found"
**Решение:**
1. Проверьте, включен ли Bluetooth на сервере Home Assistant
2. Убедитесь, что мультиварка включена и в режиме паринга
3. Проверьте MAC-адрес на опечатки

### Проблема: "Authentication failed"
**Решение:**
1. Убедитесь, что мультиварка в режиме паринга
2. Проверьте правильность пароля
3. Попробуйте перезагрузить мультиварку

## Логирование

Для диагностики проблем включите подробное логирование в `configuration.yaml`:

```yaml
logger:
  default: info
  logs:
    custom_components.skycooker: debug
```

## Поддержка

Если у вас возникли проблемы:
1. Проверьте раздел "Диагностика проблем" выше
2. Включите логирование и проверьте логи
3. Создайте issue в репозитории с описанием проблемы и логами
