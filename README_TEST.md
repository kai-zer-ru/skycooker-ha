# SkyCooker HA - Тестирование подключения

## Быстрое тестирование

### 1. Standalone тесты (рекомендуется)

Для тестирования подключения без зависимости от Home Assistant:

```bash
python3 test_connection_standalone.py
```

**Преимущества standalone тестов:**
- Проверяет наличие зависимостей
- Не требуют установки Home Assistant
- Проще в использовании
- Лучше подходят для отладки

### 2. Сканирование устройств

Запустите любой из тестовых скриптов для поиска вашей мультиварки:

```bash
python test_connection_standalone.py
# или
python test_connection.py
```

Скрипт покажет все доступные Bluetooth устройства и выделит те, что похожи на Redmond устройства (начинающиеся с "RMC-" или "RK-").

### 3. Подключение к мультиварке

Для тестирования подключения вам нужно:

1. **Найти MAC-адрес вашей мультиварки** из вывода сканирования
2. **Включить режим пары** на мультиварке (обычно удерживанием кнопки питания 3-5 секунд)
3. **Обновить параметры в скрипте**:

```python
# В файле test_connection.py замените:
mac_address = "XX:XX:XX:XX:XX:XX"  # На реальный MAC-адрес
key = [0x00, 0x01, 0x02, 0x03, 0x04, 0x05, 0x06, 0x07]  # Стандартный ключ
model = "RMC-M40S"  # Ваша модель
```

### 4. Запуск теста

```bash
python test_connection.py
```

### 5. Интерпретация результатов

**Успешное подключение:**
```
✓ Подключение успешно!
  Версия прошивки: (1, 0)
  Текущая температура: 25
  Текущий режим: None
  Целевая температура: 25
  Целевой режим: None
  Успешность подключений: 100%
```

**Ошибки подключения:**
- **"Auth failed"** - Неверный ключ аутентификации или мультиварка не в режиме пары
- **"not connected"** - Не удалось установить Bluetooth соединение
- **"Receive timeout"** - Проблемы с Bluetooth или расстояние до устройства

## Поиск MAC-адреса

Если вы не видите вашу мультиварку в списке:

1. **Проверьте Bluetooth** - Убедитесь, что Bluetooth включен на вашем сервере
2. **Включите режим пары** на мультиварке
3. **Проверьте расстояние** - Мультиварка должна быть в пределах 10 метров
4. **Попробуйте перезагрузить Bluetooth**:

```bash
sudo systemctl restart bluetooth
```

## Типичные проблемы

### 1. "No module named 'bleak_retry_connector'"

**Причина**: Модуль не установлен в системе.

**Решение для HAOS:**
```bash
apk add py3-bleak
apk add py3-bleak-retry-connector
```

**Решение для других систем:**
```bash
pip install bleak bleak-retry-connector
```

### 2. "BleakOutOfConnectionSlotsError"

**Решение:** Используйте постоянное подключение (persistent=True) как в нашем обновленном коде.

### 3. "Auth failed"

**Решение:**
1. Убедитесь, что мультиварка в режиме сопряжения
2. Попробуйте перезагрузить мультиварку
3. Проверьте MAC-адрес устройства
4. Попробуйте другой ключ аутентификации

### 4. "Device not found"

**Причина**: Мультиварка не найдена в Bluetooth.

**Решение**:
1. Убедитесь, что мультиварка включена
2. Проверьте, что она в режиме сопряжения (мигает индикатор)
3. Убедитесь, что Bluetooth адаптер работает
4. Проверьте расстояние до мультиварки

## Требования

### Для standalone тестов:
- Python 3.8+
- bleak
- bleak_retry_connector

### Для тестов в HA:
- Python 3.8+
- bleak
- bleak_retry_connector
- homeassistant

## Зависимости

### Установка в HAOS
```bash
apk add py3-bleak
apk add py3-bleak-retry-connector
```

### Установка в других системах
```bash
pip install bleak bleak-retry-connector
```

### Установка в Home Assistant
Если вы используете Home Assistant в Docker, зависимости должны быть установлены автоматически через manifest.json.

## Режимы работы

### Standalone режим
- Запускается независимо от Home Assistant
- Подходит для отладки и тестирования
- Проверяет наличие зависимостей

### HA режим
- Работает в среде Home Assistant
- Требует установки зависимостей
- Подходит для интеграции с системой
